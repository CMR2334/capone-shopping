# capone-shopping

Capital One Shopping offer tracker — a PWA that pulls cashback offers from Gmail, parses them, and surfaces them in a clean mobile-first UI with cross-device sync.

- **Live:** https://CMR2334.github.io/capone-shopping/
- **Repo:** https://github.com/CMR2334/capone-shopping
- **Source inbox:** `cmreko91@gmail.com`
- **Sender filter:** `hello@capitaloneshopping.com`

---

## How it works

```
Gmail inbox
    └── ingestor/ingest.js  (GitHub Actions, every 15 min)
            └── public/offers.json  (committed to main, served by GitHub Pages)
                    └── public/index.html  (PWA fetches & renders)
                            └── sync-worker  (Cloudflare Worker, favorites/hidden state)
```

1. GitHub Actions runs the ingestor on a 15-minute cron, parses the last 30 days of Capital One Shopping emails, and commits `public/offers.json` to `main` if anything changed.
2. The deploy job immediately publishes the `public/` folder to GitHub Pages.
3. The PWA fetches `offers.json` on load and every 5 minutes, renders offer cards, and persists favorites/hidden via the Cloudflare sync worker.

## PWA — `public/index.html`

Single self-contained file. No bundler, no dependencies, no build step.

### Features

| Feature | Detail |
|---|---|
| **Offer cards** | Merchant logo, % back, expiry countdown, activation link |
| **Sort** | Highest %, Expiring soonest, Newest (by email date) |
| **View** | All, Favorites, Hidden |
| **Favorite** | Star button; persists to localStorage + syncs across devices |
| **Hide** | X button; hides a merchant at its current %; auto-reappears if % goes higher |
| **Unhide** | Eye button in Hidden view |
| **Sync indicator** | Colored dot in header — see below |
| **Auto-refresh** | Fetches `offers.json` every 5 minutes and on tab focus |
| **Offline-ish** | Renders from last-fetched data while re-fetching in background |
| **PWA** | Installable on iOS/Android; `apple-mobile-web-app-capable`; safe-area insets |

### Sync indicator states

Sync is **always on** (see bootstrap below), so in normal use the dot is green or red — gray is rare.

| Color | Meaning |
|---|---|
| Green (glowing) | Sync token present and last `PUT /state` or `GET /state` succeeded. Normal state. |
| Red | Sync token present but last request failed (network error or 4xx/5xx). |
| Gray | No `capone:sync-token` in localStorage. Rare — every device converges on a shared token on load, so gray only shows briefly before bootstrap or if `localStorage` is blocked. |

### localStorage keys

| Key | Value |
|---|---|
| `capone:favorites` | JSON array of merchant name strings |
| `capone:hidden` | JSON object `{ merchantName: percentBack }` — threshold at hide time |
| `capone:sync-token` | Bearer token for the Cloudflare sync worker |

### Cross-device sync bootstrap

Sync is **always on**. On first load every device converges on a shared token (`SHARED_SYNC_TOKEN` in `index.html`); the worker keys state by token, so all devices read/write one KV bucket = true cross-device sync with zero setup. Legacy per-device tokens (older random-UUID or hex tokens) auto-migrate to the shared token on next load.

Opening the app with `?sync=TOKEN` overrides the shared token for a custom/private namespace — it writes the token to localStorage and cleans the URL via `history.replaceState`.

On load the app does a merge:
- If the remote has data → overwrite local.
- If the remote is empty but local has data → push local state up.

Every favorite/hide/unhide action immediately fires a `PUT /state`.

### Architecture notes (bugs fixed, don't regress)

- **Mobile grid:** uses `minmax(0, 1fr)` — **not** `1fr` (which resolves to `minmax(auto, 1fr)`). The `auto` minimum allows content to push cards wider than the viewport, causing right-side overflow. `minmax(0, 1fr)` forces the column to shrink to zero minimum.
- **Logo vertical alignment:** `.brand` has `align-self: flex-start`. Without it, the flex container stretches `.brand` to the height of the controls row, compressing the logo image.
- **Mobile action buttons:** Star/X buttons are padded to a **26px tap target** on mobile (`padding: 13px`; SVG is 26×26). Don't reduce.

## Ingestor

### `ingestor/auth.js` — one-time setup

Runs a local HTTP server on port 53117, opens the Google OAuth consent URL, captures the redirect, exchanges for a refresh token, and writes `token.json`.

```bash
npm run auth
# Opens browser → sign in with cmreko91@gmail.com → token.json written
```

If Google doesn't return a refresh token (happens when the app was previously authorized), revoke at https://myaccount.google.com/permissions and rerun.

### `ingestor/ingest.js` — main runner

```bash
npm run ingest              # uses client_secret.json + token.json locally
SAVE_RAW=1 npm run ingest   # also dumps raw HTML to ingestor/raw/ for debugging
```

Steps:
1. Authenticates via refresh token (local files or env vars).
2. Searches Gmail for emails from `hello@capitaloneshopping.com` within the last 30 days.
3. Parses each email with `parser.js`.
4. Merges offers by merchant — keeps the entry with the latest expiry; ties broken by higher %.
5. Filters out already-expired offers.
6. Writes `public/offers.json`.

`offers.json` shape:
```json
{
  "lastUpdated": "2026-05-14T12:00:00.000Z",
  "sourceCount": 42,
  "parseFailures": 0,
  "offers": [
    {
      "merchant": "Nike",
      "percentBack": 10,
      "wasPercent": 5,
      "expiresAt": "2026-05-20T07:00:00.000Z",
      "activationUrl": "https://...",
      "logoUrl": "https://...",
      "source": "single-use",
      "emailMessageId": "...",
      "emailDate": "2026-05-13T..."
    }
  ],
  "upcomingReveals": []
}
```

`source` values:
- `single-use` — standard offer from a list email (has its own expiry)
- `todays-top` — featured "Today's Top" merchant (uses the "Rewards offer ends on" date)

### `ingestor/parser.js`

Cheerio-based HTML parser. Pattern matched against DOM text nodes:

```
(Earn )?{N}% back at {Merchant}
```

- Emails with `Earn` → `single-use`; without → `todays-top`
- Expiry parsed from header text: `Expiring {Month} {Day}` or `Rewards offer ends on {Month} {Day}`
- Activation link: nearest ancestor `<a href>` or sibling link within 8 hops up the DOM
- Logo: nearest `<img src>` within 8 hops up the DOM
- Deduplication within a single email by merchant name

## Sync Worker — `sync-worker/`

Cloudflare Worker. Stores per-token state in a KV namespace (`OFFERS_KV`).

**Worker URL:** `https://capone-offers-sync.cmr2334.workers.dev`

### Endpoints

| Method | Path | Auth | Description |
|---|---|---|---|
| `GET` | `/state` | Bearer token | Returns `{ favorites, hidden, updatedAt }` |
| `PUT` | `/state` | Bearer token | Writes `{ favorites, hidden }` |
| `GET` | `/action` | `?token=` query param | Favorite/hide/unhide action for email links; returns HTML confirmation |
| `GET` | `/health` | None | Returns `ok` |

`/action` query params: `type` (`favorite`, `hide`, `unhide`), `merchant`, `percent` (for hide).

CORS allows `https://cmr2334.github.io` and localhost variants.

### Deploy

```bash
cd sync-worker
wrangler kv namespace create OFFERS_KV   # paste the id into wrangler.toml
wrangler secret put SYNC_TOKEN           # random hex string — share this in ?sync=TOKEN URLs
wrangler deploy
```

## GitHub Actions — `.github/workflows/ingest.yml`

Runs on:
- Push to `main`
- Schedule: every 15 minutes (`*/15 * * * *`)
- Manual dispatch

Two jobs:

**`ingest`**
1. Checks out repo, installs Node 20 deps.
2. Runs `node ingestor/ingest.js` with Gmail creds from repo secrets.
3. If `public/offers.json` changed: commits with message `ingest: refresh offers {timestamp} [skip ci]` and pushes.

**`deploy`** (needs `ingest`)
1. Uploads `public/` as a Pages artifact.
2. Deploys to GitHub Pages.

Required repo secrets:

| Secret | Value |
|---|---|
| `GMAIL_CLIENT_ID` | OAuth client ID from `client_secret.json` |
| `GMAIL_CLIENT_SECRET` | OAuth client secret |
| `GMAIL_REFRESH_TOKEN` | Refresh token from `token.json` after running `npm run auth` |

## First-time setup

### 1. Google OAuth

```bash
# Put client_secret.json (Google Cloud OAuth desktop app) in project root
npm install
npm run auth
# Sign in with cmreko91@gmail.com — token.json is written
```

### 2. GitHub secrets

Copy `GMAIL_CLIENT_ID`, `GMAIL_CLIENT_SECRET` from `client_secret.json`, and `GMAIL_REFRESH_TOKEN` from `token.json` into the repo's Settings → Secrets → Actions.

### 3. Sync worker (optional)

```bash
cd sync-worker
wrangler kv namespace create OFFERS_KV
# Update kv_namespaces id in wrangler.toml
wrangler secret put SYNC_TOKEN   # any random string, e.g. openssl rand -hex 32
wrangler deploy
```

Open the live site with `?sync=YOUR_TOKEN` to activate sync on a device.

### 4. GitHub Pages

Enable Pages in repo Settings → Pages → Source: GitHub Actions. The workflow handles deployment.
